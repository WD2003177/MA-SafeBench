MA-SafeBench server run commands
================================

This file is for running MA-SafeBench on the remote server with CARLA 0.9.13.
It uses the same CARLA port style as your ChatScene run:

- CARLA path: /home/ubuntu/wudi/carla_0.9.13
- SafeBench path: /home/ubuntu/wudi/ma-safebench
- CARLA port: 2402
- Traffic-manager port: 8402

If the repo directory name is different on the server, only change SAFE_ROOT.


0. One-time environment check
-----------------------------

conda activate safebench

export SAFE_ROOT=/home/ubuntu/wudi/ma-safebench
export CARLA_ROOT=/home/ubuntu/wudi/carla_0.9.13

cd "$SAFE_ROOT"
pip install -r requirements.txt
pip install -e .


1. Terminal A: start CARLA
--------------------------

# If the tmux session already exists:
tmux attach -t carla0913

# If it does not exist:
# tmux new -s carla0913

export CARLA_ROOT=/home/ubuntu/wudi/carla_0.9.13
cd "$CARLA_ROOT"

./CarlaUE4.sh -prefernvidia -RenderOffScreen -nosound -carla-port=2402 -quality-level=Low


2. Terminal B: run MA-SafeBench, single route first
--------------------------------------------------

conda activate safebench

export SAFE_ROOT=/home/ubuntu/wudi/ma-safebench
export CARLA_ROOT=/home/ubuntu/wudi/carla_0.9.13

export PYTHONPATH="${SAFE_ROOT}:${CARLA_ROOT}/PythonAPI:${CARLA_ROOT}/PythonAPI/carla:${CARLA_ROOT}/PythonAPI/carla/agents:${CARLA_ROOT}/PythonAPI/carla/dist/carla-0.9.13-py3.8-linux-x86_64.egg:${PYTHONPATH}"

# OpenAI-compatible LLM settings. Do not write the real key into git.
export MA_LLM_API_KEY="sk-b8da07d06a224a8f9ff1e72dff94fb06"
export MA_LLM_MODEL="deepseek-chat"
export MA_LLM_BASE_URL="https://api.deepseek.com"
export MA_LLM_TIMEOUT_S=45

cd "$SAFE_ROOT"

SDL_VIDEODRIVER=dummy python scripts/run.py \
  --agent_cfg sac.yaml \
  --scenario_cfg ma_cut_in.yaml \
  --mode eval \
  --num_scenario 1 \
  --max_episode_step 300 \
  --save_video \
  --port 2402 \
  --tm_port 8402 \
  --fixed_delta_seconds 0.1 \
  --exp_name ma_cut_in_s3_r0_llm \
  --output_dir log

Notes:
- ma_cut_in.yaml currently selects scenario_id: 3 and route_id: 0.
- scripts/run.py does not expose scenario_id/route_id as CLI arguments, so change
  safebench/scenario/config/ma_cut_in.yaml if you want another route.
- Do not add "--render False": this repo parses it with type=bool, so the string
  "False" is still treated as True. Use SDL_VIDEODRIVER=dummy for headless runs.


3. Terminal B: full route evaluation
------------------------------------

After the single-route command works, run all routes for scenario 3:

cd "$SAFE_ROOT"

SDL_VIDEODRIVER=dummy python scripts/run.py \
  --agent_cfg sac.yaml \
  --scenario_cfg ma_cut_in_all_routes.yaml \
  --mode eval \
  --num_scenario 1 \
  --max_episode_step 300 \
  --save_video \
  --port 2402 \
  --tm_port 8402 \
  --fixed_delta_seconds 0.1 \
  --exp_name ma_cut_in_s3_all_routes_llm \
  --output_dir log

ma_cut_in_all_routes.yaml currently selects scenario_id: 3 and route_id: null,
so SafeBench will evaluate all route entries for scenario 3.


4. Useful outputs
-----------------

SafeBench creates logs under:

log/<exp_name>/<exp_name>_sac_ma_seed_0/

Important files:

- eval_results/results.pkl
- eval_results/records.pkl
- ma_trace_env_0.jsonl
- config.json / config.yaml

The MA trace JSONL stores raw LLM decisions, contracts, verifier status, planned
behaviors, and MA event metrics for the run.


5. If you want the no-LLM rule fallback ablation
------------------------------------------------

Use the same command as section 2, but unset the key and change exp_name:

unset MA_LLM_API_KEY

SDL_VIDEODRIVER=dummy python scripts/run.py \
  --agent_cfg sac.yaml \
  --scenario_cfg ma_cut_in.yaml \
  --mode eval \
  --num_scenario 1 \
  --max_episode_step 300 \
  --save_video \
  --port 2402 \
  --tm_port 8402 \
  --fixed_delta_seconds 0.1 \
  --exp_name ma_cut_in_s3_r0_rule_fallback \
  --output_dir log
